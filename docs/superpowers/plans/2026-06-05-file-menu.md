# File Menu Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`).

**Goal:** Populate the File menu with Open Folder / Open File / Open Recent / Close Editor / Close Folder / Close Window (New Window deferred), wired to airlock's current read-only capabilities.

**Architecture:** menu.ts builds the File submenu and pushes a discriminated `menu:action` to the focused window on each click; a renderer subscriber dispatches to store actions / IPCs. Recent folders persist in `AppPrefs.recentFolders` and drive the Open Recent submenu (menu rebuilt via `applyAppMenu`, now taking a 3rd `recentFolders` param). New main IPCs: `workspace:open`, `workspace:close`, `dialog:openFile`; `dialog:openFolder` records recents.

**Tech Stack:** Electron Menu/dialog, TypeScript (strict, noUncheckedIndexedAccess), React/Zustand, vitest, biome.

**Spec:** `docs/superpowers/specs/2026-06-05-file-menu-design.md`

**Constraints:** ASCII-only in main/shared/preload (menu.ts, ipc.ts, prefs.ts, index.ts, shared/ipc.ts, preload) -- menu labels use `"..."` not an ellipsis glyph. Renderer exempt. No secret-value surface touched.

```ts
const RECENT_CAP = 10;
function sanitizeRecentFolders(raw: unknown): string[] {
  if (!Array.isArray(raw)) return [];
  const seen = new Set<string>();
  const out: string[] = [];
  for (const v of raw) {
    if (typeof v === "string" && v.length > 0 && !seen.has(v)) {
      seen.add(v);
      out.push(v);
      if (out.length >= RECENT_CAP) break;
    }
  }
  return out;
}
```

Inside `registerIpc` (so it closes over `prefsFile` + `onFolderOpen`), add:
```ts
  // Open a workspace at a known path: set root, re-register MCP, record the
  // folder in recents, and rebuild the menu so Open Recent reflects it.
  async function recordAndOpen(root: string): Promise<void> {
    workspaceRoot = root;
    onFolderOpen?.(root);
    const prev = await loadPrefs(prefsFile);
    const recents = [root, ...prev.recentFolders.filter((p) => p !== root)].slice(
      0,
      RECENT_CAP,
    );
    await savePrefs(prefsFile, { recentFolders: recents });
    applyAppMenu(prefsFile, prev.sectionVisibility, recents);
  }
```

- [ ] **Step 2: record recents in dialog:openFolder.** Change the existing handler so the picked folder routes through `recordAndOpen` (replacing the inline `workspaceRoot = ...; onFolderOpen?.(...)`):
```ts

    const r = await dialog.showOpenDialog({ properties: ["openDirectory"] });
    if (r.canceled || r.filePaths.length === 0) return null;
    const picked = r.filePaths[0];
    if (!picked) return null;
    await recordAndOpen(picked);
    return picked;

```

- [ ] **Step 3: workspace:open / workspace:close / dialog:openFile.** Add inside registerIpc:
```ts
  ipcMain.handle("workspace:open", async (_e, p: unknown) => {
    if (typeof p !== "string") throw new Error("Invalid payload");
    await recordAndOpen(p);
    return p;
  });

  ipcMain.handle("workspace:close", () => {
    workspaceRoot = null;
  });

  // Pick a file to view; return it RELATIVE to the open folder (the viewer's
  // read path is workspace-confined). null if cancelled, no folder, or outside.
  ipcMain.handle("dialog:openFile", async () => {
    if (!workspaceRoot) return null;
    const r = await dialog.showOpenDialog({
      properties: ["openFile"],
      defaultPath: workspaceRoot,
    });
    if (r.canceled || r.filePaths.length === 0) return null;
    const picked = r.filePaths[0];
    if (!picked) return null;
    const rel = path.relative(workspaceRoot, picked);
    if (rel.startsWith("..") || path.isAbsolute(rel)) return null; // outside root
    return rel;
  });
```
(`path` is already imported in ipc.ts; confirm. If not, `import path from "node:path"` matching the file's style.)

- [ ] **Step 1: the dispatcher hook.** Create `useMenuActions.ts`:
```ts
import { useEffect } from "react";
import { useApp } from "../store";

export function useMenuActions(): void {
  useEffect(() => {
    return window.airlock.onMenuAction(async (a) => {
      const s = useApp.getState();
      switch (a.type) {
        case "open-folder": {
          const picked = await window.airlock.openFolder();
          if (picked) s.setRoot(picked);
          break;
        }
        case "open-recent": {
          const root = await window.airlock.workspaceOpen(a.path);
          if (root) s.setRoot(root);
          break;
        }
        case "open-file": {
          const rel = await window.airlock.openFile();
          if (rel) {
            const file = await window.airlock.readFile(rel);
            s.setSelected(rel, file);
          }
          break;
        }
        case "close-editor": {
          s.setDiff(null);
          s.setSettingsOpen(false);
          s.setDbView(null);
          s.setSelected(null, null);
          break;
        }
        case "close-folder": {
          await window.airlock.workspaceClose();
          s.setRoot(null);
          break;
        }
      }
    });
  }, []);
}
```

## Self-review notes
- New Window deferred (multi-window refactor); editing items deferred (read-only viewer) -- documented.
- Open File is within-workspace only (the confined read path); the dialog returns a root-relative path, null if outside/none.
- recents recorded in one place (recordAndOpen) used by both dialog:openFolder + workspace:open; menu rebuilt with recents on every open + on visibility change.
- Cmd+W -> Close Editor, Cmd+Shift+W -> Close Window (matches VS Code); Close Editor/Folder are graceful no-ops when nothing is open.
- ASCII-only menu labels ("..."); no secret-value surface touched.
- ipc.ts -> menu.ts import is one-way (menu.ts imports only prefs/shared) -- no cycle.
